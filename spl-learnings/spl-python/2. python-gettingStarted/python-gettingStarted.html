Verify Python is already Installed on your system?

To verify if Python is installed on your system, and to check its version, you can use the following steps:

For Windows:

    1. Open the Command Prompt: Press Win + R, type "cmd", and press Enter.

    2. Type the following command and press Enter:

      1. python --version	# This will display the installed Python version if it's already installed..

      2. python			# To enter the Python interpreter.

    If Python is already installed then you will get a message like below

      $ python
      Python 3.6.8 (default, Sep 10 2021, 09:13:53)
      [GCC 8.5.0 20210514 (Red Hat 8.5.0-3)] on linux
      Type "help", "copyright", "credits" or "license" for more information.
      >>>

    if you don't see output like above then you need to Install the python installer software on your system.

For macOS and Linux:

    1. Open terminal.usually terminal application can be found in your applications or by searching for "Terminal" in the search bar.

    2. Type the following command and press Enter:

      python --version		# This will display the installed Python version if it's already installed.
      python					# To enter the Python interpreter.

    You should see the Python prompt, indicating that Python is installed. if not then you need to Install the python installer software on your system


Python Installation:

To install Python on your system, follow these steps:

For Windows:

    1. Download Python Installer:

      Go to the official Python website at Python Installer for Windows

      You'll see the latest version of Python listed. Make sure to download the Python 3.x version (e.g., Python 3.9.6) as Python 2.x is no longer supported.

    2. Run the Installer:

      Double-click the downloaded installer to run it. On the initial installation screen, make sure to check the box that says "Add Python x.x to PATH." This option is essential for easily running Python from the command line.

    3. (Optional) Customize Installation:

      Click on the "Customize installation" button if you want to customize the installation directory or select optional features. Most users can leave the defaults selected.

    4. Install Python:

      Click the "Install Now" button to start the installation. The installer will copy Python files to your system.

    5. Verify the Installation:

      Once the installation is complete, open the Command Prompt by pressing Win + R, typing "cmd," and pressing Enter. Type the following command to verify that Python is installed and check its version: python --version

      If Python is installed, you'll see the python version number. If not, you'll receive an error message..

    6. Testing Python:

      To test Python, you can enter the Python interactive shell. Type python and press Enter. You should see the Python prompt (>>>) indicating that you are in the Python shell.

      To exit the Python shell, simply type exit() and press Enter.

    Congratulations! You have successfully installed Python on your Windows system. You can now start using Python for scripting and programming tasks. Remember to keep Python and any installed packages up to date for the best experience.

For Linux:

    many Linux distributions come with Python pre-installed. 

      #On Debian/Ubuntu
      sudo apt install python3

      #On Red Hat/CentOS
      sudo yum install python3

For Mac:

    macOS typically comes with a version of Python pre-installed (usually Python 2.x). It's recommended to use Python 3.x for new projects as Python 2 is no longer maintained.

complete python installation docs can be found here https://docs.python.org/3/using/index.html

virtual environment steup using venv[in-built]

A virtual environment, often abbreviated as "virtual env" or "venv," is a self-contained environment within a computer system that allows you to install and manage software packages and libraries independently of the system's global environment. Virtual environments are commonly used in software development to isolate project-specific dependencies, ensuring that different projects can have their own set of libraries and packages without interfering with each other.


key points about virtual environments

    1. Isolation: Virtual environments provide isolation for Python or other programming language installations and libraries. This means you can have different versions of libraries or packages installed for different projects, preventing conflicts.

    2. Dependency Management: You can easily manage project-specific dependencies within a virtual environment. You can install, update, and remove packages as needed without affecting the global system or other projects.

    3. Portability: Virtual environments are portable. You can create a virtual environment on one system, and as long as you have the environment configuration saved (typically in a requirements.txt file), you can recreate the same environment on another system.

    4. Version Compatibility: Virtual environments help ensure version compatibility between packages. You can specify the exact versions of libraries your project requires.

    5. Cleaner Development: Using virtual environments keeps your development environment clean and avoids cluttering your global Python installation with project-specific packages.


Typical workflow for working with virtual environments in Python

    1. Create a virtual environment for your project.

    2. Activate the virtual environment.

    3. Install the required packages using pip within the virtual environment.

    4. Develop and run your Python project within the activated virtual environment.

    5. When done, deactivate the virtual environment.

Virtual environments are a fundamental tool for Python developers to manage dependencies and maintain project-specific environments, making it easier to develop and distribute Python projects without worrying about conflicts or dependencies on the system-wide Python installation.

    1. Create a virtual environment in a directory (e.g., 'myenv')

    	>> python -m venv myenv

    2. Activate the virtual environment

    	>> myenv\Scripts\activate	# on Windows
       	>> source myenv/bin/activate	# on macOS and Linux

      You need to activate a virtual environment to use it. When activated, it changes the shell's PATH environment variable to point to the Python interpreter and packages within the virtual environment. This allows you to use the isolated environment for running your Python scripts and applications.


    3. Install Required packages within the virtual environment using 'pip'

    	>> pip install package-name

    4. Deactivate the virtual environment

    	>> deactivate

    Note:

      - don't forget to deactivate the virtual environment when you're finished work to return to the system-wide(global) Python environment.

      - Remember that virtual environments are not limited to Python. Similar concepts and tools exist for other programming languages, such as 'conda' for managing environments in the data science and scientific computing fields. Virtual environments are a best practice in software development for maintaining clean, reproducible, and isolated project environments.
